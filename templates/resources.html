Splunk Search Syntax & Examples

Typical Search Syntax:

index=[index name] sourcetype=[source type name] [search keyword(s)]

Common Search Examples:

Search Type Splunk Query
Production D2AP Device/FW Logs index=device_prod sourcetype="kube:container:backend-log-server" SS=FW
Production Basestation Connected Device/FW Logs index=xc_services_prod sourcetype="kube:container:metric-collector"
Production Client/App iOS Logs index=device_prod sourcetype="kube:container:backend-log-server" SS=IOS
Production Client/App Android Logs index=device_prod sourcetype="kube:container:backend-log-server" SS=ANDROID
Production Web App (Front End) Logs index=device_prod sourcetype="kube:container:backend-log-server" SS=FE
Product Savant / Object Detection Logs index=savant_prod sourcetype="var:log:syslog"
QA Logs (by Camera ID or Device ID) index=*qa cameraid="4CD161SJ06321" OR deviceid="48B2557L48BE6"

Pro Tip: Replace camera ID and device ID with actual values when searching QA logs.

💡 Pro Tips

▼
  • Bookmark your most-used runbooks for quick access during incidents
  • Create custom Jira filters for your common queries
  • Set up saved searches in ServiceNow for recurring tasks
  • Use Slack reminders to follow up on pending items
  • Keep a personal wiki page with your notes and learnings
  • Subscribe to relevant Confluence spaces for automatic updates
  • Save frequently used Splunk searches - Click "Save As" in Splunk to create alerts or scheduled reports
  • Use Splunk time range shortcuts - earliest=-24h, earliest=-7d, or earliest=@d (today) for faster searches
